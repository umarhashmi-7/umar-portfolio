Offline-First Notes

Rich-text Markdown editor with full folder structure, tag matching, and local search. Instant boot speed using Room caching.

Kotlin Markdown Room DB Flow API
60 FPS Typing Speed
< 15ms FTS4 Query

Designed on clean architecture principles utilizing SQLite FTS4 virtual tables, a custom MarkdownParser.kt compiler, and Kotlin Coroutine Flow stream handlers.

Searching thousands of markdown bodies on the UI thread caused noticeable micro-stutters during typing.

Utilized SQLite's FTS4 extension within Room database and routed text changes through a 250ms debounced Kotlin Flow.

Impact & Results: Shipped modular search structures to production pipelines, securing smooth editor loads on heavy documents.

Lessons Learned: Taught the value of offloading complex search tasks to worker threads using Coroutines. Choosing FTS4 index files slightly increased local disk usage but yielded a 90% reduction in query latency on large document lists.
